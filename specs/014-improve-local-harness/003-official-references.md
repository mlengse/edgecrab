## OpenAI API semantics (compatibility baseline)

| Topic | Source | Implication |
|-------|--------|-------------|
| `max_tokens` | [Chat create — max_tokens](https://platform.openai.com/docs/api-reference/chat/create) | Caps **generated** tokens only |
| `finish_reason: length` | [Chat object — choices](https://platform.openai.com/docs/api-reference/chat/object#chat/object-choices) | Generation stopped at cap — not an transport error |
| Tool calls in response | [Function calling guide](https://platform.openai.com/docs/guides/function-calling) | `tool_calls[].function.arguments` is a **JSON string** generated by the model |
| Reasoning + completion budget | [Community: O4-mini empty + length](https://community.openai.com/t/o4-mini-returns-empty-response-because-reasoning-token-used-all-the-completion-token/1359002) | Reasoning tokens can consume entire `max_completion_tokens`, leaving empty content |
| Function JSON truncation | [Community: max_tokens + functions](https://community.openai.com/t/issue-with-max-tokens-in-case-of-using-functions/434887) | Truncated `arguments` → invalid JSON → no usable tool call |

**OpenAI `reasoning_effort`:** Documented on Chat Completions create ([API ref](https://platform.openai.com/docs/api-reference/chat/create)); values include `none` on supported models. EdgeCrab sends this via edgequake-llm; LM Studio may map to template kwargs depending on build.

## What official docs do **not** guarantee

1. **Sub-second tool turns** at 30k+ prompt — not specified; local latency is hardware-bound.
2. **Valid tool_calls** when generation hits `max_tokens` mid-JSON — opposite: truncation is expected ([OpenAI community](https://community.openai.com/t/issue-with-max-tokens-in-case-of-using-functions/434887)).
3. **`reasoning_effort: none`** on every local server — Qwen may need template kwargs ([Doubleword](https://docs.doubleword.ai/inference-api/models/qwen-qwen3-5-9b)); EdgeCrab must verify via `thinking_tokens` in logs.
4. **Automatic parent directory creation** on write — filesystem concern, not LLM API; EdgeCrab `write_file` requires `create_dirs: true` explicitly.

These gaps are **harness responsibilities**, not server bugs.
